# resource-based_economy_strategy

Симуляция ресурсно-ориентированной экономики в игровой форме.

Версия `0.0.6` уточняет первый игровой слой: старт теперь всегда идёт от
единственного бесплатного центра города, одна точка принимает только одну
постройку, а меню учитывает биом и рельеф (шахта доступна только в горах).
Склад больше не использует ресурс `wood`; брёвна и уголь расходуются на
энергию в `МВт·сут`, еда и вода считаются в реалистичных долях тонны на
человека в день. WebGL-клиент получил подпись `Неделя / Месяц / Год` с
четырёхнедельным месяцем, ограничение зума у поверхности, скрытие точек
обратной стороны планеты, слой меридианов и параллелей, легенду биомов,
иконки построек и подробное описание выбранной точки с климатом.

Версия `0.0.5` добавляет первый игровой слой поверх WebGL-планеты:
биомы теперь генерируются непрерывным сферическим шумом с океанами,
материками, редкими островами, горами, озёрами и реками; статический
клиент получил вход под ником, паузу и недельный тик раз в 5 секунд,
выбор точки на максимальном LOD, меню планирования построек, склад,
демографию и строку событий. В ядре появились индивидуальные жители
`Чел1`, `Чел2` с base32-id, взросллением в 16 лет, смертями от
жажды, голода, холода или старости, стартом игрока с 10 людьми и
запасом еды/воды на игровой год.

Версия `0.0.4` уходит с 2D-канвы на настоящий 3D: появился
WebGL2-просмотрщик планеты, рисующий её как сферическое облако точек,
полученное Фибоначчиевой решёткой. Ладдер LOD теперь оперирует точками
(`2 000 → 20 000 → 200 000`) при логических 10 000 000 на сервере — к
каждой точке в будущих версиях можно будет привязать 3D-модель здания
или подвижного транспорта, прорисовываемую только на нужном зуме.
Управление и пороги переключения LOD совпадают с 0.0.3.

Версия `0.0.3` улучшает планетарный просмотрщик: к мыши и колесу
добавилось управление с клавиатуры (`WASD` для вращения, `QE` для крена,
`+`/`-` для зума, `R` для сброса), а одно фиксированное разрешение
сменилось ладдером LOD из трёх ISEA3H-уровней (`92 → 812 → 7292` ячеек).
Активный уровень переключается по текущему зуму, поэтому при приближении
клиент рисует более детальную сферу, а при отдалении — лёгкий меш.

Версия `0.0.2` добавляет первый планетарный слой для будущего клиента:

- логическая сетка `ISEA3H` с aperture 3 и оценкой разрешения под гекс
  около 10 метров на планете радиусом в 10 раз меньше Земли;
- процедурный полигональный меш сферы из гексагонов и 12 обязательных
  пентагонов без материализации миллиардов ячеек;
- чанки, пространственный индекс и формат событий вида
  `игрок начал стройку в #<id>`, чтобы передавать действия, а не полигоны;
- статический HTML-просмотрщик, где шар можно вращать и масштабировать.

Базовое проверяемое Python-ядро симуляции сохраняет механику версии `0.0.1`:

- нет денег, валюты и рыночных цен, только ресурсы;
- один такт симуляции равен одному игровому дню;
- люди ежедневно потребляют еду и воду, а в холоде требуют энергию;
- здания производят, потребляют и перерабатывают ресурсы через рецепты;
- перенос ресурсов ограничен дневной логистической емкостью людей;
- технология колеса увеличивает логистику после появления досок и круглого
  леса;
- погода влияет на солнечную, ветровую и водную генерацию;
- `fast_forward()` пропускает несколько дней и возвращает статистику
  удовлетворения потребностей;
- управляемый запуск игры идёт через один файл входа:
  `python -m resource_based_economy_strategy`;
- игра завершается при смерти всех жителей или при вводе пользователем `0`;
- генерация стартового мира может учитывать зерно `--seed`.

## Запуск

Проект не требует внешних Python-зависимостей.

```bash
python -m resource_based_economy_strategy --days 30 --people 6 --seed 42
```

Для автоматического прогона без ожидания ввода:

```bash
python -m resource_based_economy_strategy --days 30 --people 6 --seed 42 --auto
```

Дополнительные демонстрации:

```bash
python examples/run_day_simulation.py
python examples/run_hex_sphere_viewer.py
python examples/run_webgl_planet_viewer.py
python examples/run_planet_tour.py
python examples/run_time_control.py
```

После генерации просмотрщик открывается как обычный файл:
[`examples/hex_sphere_viewer.html`](examples/hex_sphere_viewer.html) —
2D-канва с гекс-сферой версии 0.0.3, и
[`examples/webgl_planet_viewer.html`](examples/webgl_planet_viewer.html) —
WebGL2-просмотрщик сферического облака точек и игрового слоя версии 0.0.6.
